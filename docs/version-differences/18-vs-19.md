# React 18 vs React 19 - Key Differences

React 19 focuses on **developer experience** and **form handling**, making common patterns simpler while building on React 18's concurrent foundation.

## 🎯 TL;DR - What Changed?

React 19 makes forms easier, removes boilerplate (like `forwardRef`), and adds native support for document metadata and optimistic UI.

## Major Changes

### 1. Actions - Built-in Async State Management

**React 18 Pattern:**
```jsx
function Form() {
  const [pending, setPending] = useState(false);
  const [error, setError] = useState(null);

  async function handleSubmit(e) {
    e.preventDefault();
    setPending(true);
    setError(null);

    try {
      const formData = new FormData(e.target);
      await submitData(formData);
      e.target.reset();
    } catch (err) {
      setError(err.message);
    } finally {
      setPending(false);
    }
  }

  return (
    <form onSubmit={handleSubmit}>
      <input name="title" />
      <button disabled={pending}>
        {pending ? 'Submitting...' : 'Submit'}
      </button>
      {error && <p>{error}</p>}
    </form>
  );
}
```

**React 19 with Actions:**
```jsx
function Form() {
  const [state, formAction, isPending] = useActionState(
    async (previousState, formData) => {
      try {
        await submitData(formData);
        return { success: true };
      } catch (err) {
        return { error: err.message };
      }
    },
    null
  );

  return (
    <form action={formAction}>
      <input name="title" />
      <SubmitButton />  {/* Reads pending state from form context */}
      {state?.error && <p>{state.error}</p>}
    </form>
  );
}

function SubmitButton() {
  const { pending } = useFormStatus();
  return (
    <button type="submit" disabled={pending}>
      {pending ? 'Submitting...' : 'Submit'}
    </button>
  );
}
```

**Benefits:**
- No manual `pending` state
- No `e.preventDefault()`
- Built-in error handling pattern
- `useFormStatus` reads parent form state (no prop drilling!)

### 2. `useOptimistic` - Instant UI Updates

**React 18 Pattern:**
```jsx
function TodoList({ todos }) {
  const [optimisticTodos, setOptimisticTodos] = useState(todos);

  async function handleToggle(id) {
    // Manually manage optimistic state
    setOptimisticTodos(prev =>
      prev.map(t => t.id === id ? { ...t, completed: !t.completed } : t)
    );

    try {
      await api.toggle(id);
    } catch (err) {
      // Revert on error
      setOptimisticTodos(todos);
    }
  }

  return <List todos={optimisticTodos} onToggle={handleToggle} />;
}
```

**React 19 with `useOptimistic`:**
```jsx
function TodoList({ todos }) {
  const [optimisticTodos, addOptimistic] = useOptimistic(
    todos,
    (currentTodos, toggledId) =>
      currentTodos.map(t =>
        t.id === toggledId ? { ...t, completed: !t.completed } : t
      )
  );

  async function handleToggle(id) {
    startTransition(async () => {
      addOptimistic(id);    // Instantly update UI
      await api.toggle(id); // When the transition finishes,
                             // optimistic state reverts to base state
    });
  }

  return <List todos={optimisticTodos} onToggle={handleToggle} />;
}
```

**Benefits:**
- Optimistic value automatically reverts to the real state when the transition completes
- Simpler API than manual optimistic state management
- Works seamlessly with Actions and transitions

### 3. `ref` as a Prop - No More `forwardRef`

**React 18:**
```jsx
import { forwardRef } from 'react';

const SearchBar = forwardRef(function SearchBar(props, ref) {
  return <input ref={ref} {...props} />;
});

// Parent
function App() {
  const inputRef = useRef();
  return <SearchBar ref={inputRef} />;
}
```

**React 19:**
```jsx
// No forwardRef needed!
function SearchBar({ ref, ...props }) {
  return <input ref={ref} {...props} />;
}

// Parent - exactly the same
function App() {
  const inputRef = useRef();
  return <SearchBar ref={inputRef} />;
}
```

**Benefits:**
- Less boilerplate
- `ref` is just a regular prop
- `forwardRef` still works for backwards compatibility

### 4. Ref Cleanup Functions

**React 18:**
```jsx
function Component() {
  const ref = useCallback((node) => {
    if (node) {
      node.addEventListener('scroll', handleScroll);
    } else {
      // Can't clean up here - node is null
    }
  }, []);

  return <div ref={ref} />;
}
```

**React 19:**
```jsx
function Component() {
  const ref = useCallback((node) => {
    if (node) {
      node.addEventListener('scroll', handleScroll);

      // Return cleanup function
      return () => {
        node.removeEventListener('scroll', handleScroll);
      };
    }
  }, []);

  return <div ref={ref} />;
}
```

**Benefits:**
- Simpler cleanup logic
- Matches `useEffect` cleanup pattern
- No need to check if node is null in cleanup

### 5. Context Without `.Provider`

**React 18:**
```jsx
const ThemeContext = createContext();

function App() {
  return (
    <ThemeContext.Provider value={{ theme: 'dark' }}>
      <Page />
    </ThemeContext.Provider>
  );
}
```

**React 19:**
```jsx
const ThemeContext = createContext();

function App() {
  return (
    <ThemeContext value={{ theme: 'dark' }}>
      <Page />
    </ThemeContext>
  );
}
```

**Benefits:**
- Less typing
- Cleaner JSX
- `.Provider` still works for compatibility

### 6. `use()` Hook - Read Resources Conditionally

**React 18:**
```jsx
// ❌ Can't call useContext conditionally
function Component({ showTheme }) {
  if (showTheme) {
    const theme = useContext(ThemeContext);  // Error!
  }
}

// Must call unconditionally
function Component({ showTheme }) {
  const theme = useContext(ThemeContext);

  if (showTheme) {
    return <div>Theme: {theme}</div>;
  }
  return null;
}
```

**React 19:**
```jsx
import { use } from 'react';

// ✅ Can call use() conditionally!
function Component({ showTheme }) {
  if (showTheme) {
    const theme = use(ThemeContext);
    return <div>Theme: {theme}</div>;
  }
  return null;
}

// Also works with promises (promise must be created outside render)
const dataPromise = fetchData();

function Component() {
  const data = use(dataPromise);  // Suspends until resolved
  return <div>{data}</div>;
}
```

**Benefits:**
- More flexible than `useContext`
- Works in conditionals, loops, early returns
- Can unwrap promises (Suspense boundary required)

### 7. Document Metadata Support

**React 18 (with react-helmet):**
```jsx
import { Helmet } from 'react-helmet';

function Page() {
  return (
    <>
      <Helmet>
        <title>My Page</title>
        <meta name="description" content="Page description" />
      </Helmet>
      <div>Content</div>
    </>
  );
}
```

**React 19 (Native Support):**
```jsx
function Page() {
  return (
    <>
      <title>My Page</title>
      <meta name="description" content="Page description" />
      <div>Content</div>
    </>
  );
}
```

React automatically hoists `<title>`, `<meta>`, and `<link>` to `<head>`.

**Benefits:**
- No external library needed
- Works with streaming SSR
- Simpler mental model

---

### 8. Asset Loading APIs

**React 19 adds special handling for:**

```jsx
// Stylesheets with priority
<link rel="stylesheet" href="style.css" precedence="default" />

// Preloading resources
<link rel="preload" as="script" href="script.js" />

// Async scripts
<script async src="analytics.js" />
```

React deduplicates these and ensures correct loading order.

## New Hooks Summary

| Hook | Purpose |
|------|---------|
| `useActionState` | Form actions with pending state + error handling |
| `useFormStatus` | Read parent form's pending state |
| `useOptimistic` | Optimistic UI updates that revert when the transition completes |
| `use()` | Read context/promises, works conditionally |

## Breaking Changes

### 1. Removed: Legacy Context

```jsx
// ❌ No longer supported
class Parent extends Component {
  getChildContext() {
    return { theme: 'dark' };
  }
}

// ✅ Use modern Context API
const ThemeContext = createContext();
```

### 2. Removed: String Refs

```jsx
// ❌ No longer supported
<div ref="myDiv" />

// ✅ Use callback or object refs
<div ref={this.myRef} />
```

### 3. Removed: Module Pattern Components

```jsx
// ❌ No longer supported
function Component() {
  return {
    render() {
      return <div />;
    }
  };
}

// ✅ Return JSX directly
function Component() {
  return <div />;
}
```

### 4. Stricter Error Handling

React 19 changed its default error handling: uncaught errors that were previously swallowed in production now bubble up to the nearest Error Boundary (or `window.onerror` if none exists). Adding Error Boundaries is strongly recommended for production apps.

## Migration Checklist

### Step 1: Update Dependencies

```json
{
  "dependencies": {
    "react": "^19.0.0",
    "react-dom": "^19.0.0"
  }
}
```

### Step 2: Remove Deprecated Patterns

```bash
# Search for:
- forwardRef  # Can be replaced with direct ref prop
- .Provider   # Can be replaced with Context directly
- useContext  # Consider use() for conditional cases
```

### Step 3: Adopt Actions for Forms

Replace manual form handling with `useActionState`:

```jsx
// Before
const [loading, setLoading] = useState(false);
async function handleSubmit(e) {
  e.preventDefault();
  setLoading(true);
  // ...
}

// After
const [state, formAction] = useActionState(async (prev, formData) => {
  // ...
}, null);
```

### Step 4: Add Optimistic Updates

Replace manual optimistic state with `useOptimistic`:

```jsx
const [optimisticState, addOptimistic] = useOptimistic(
  actualState,
  (current, optimisticValue) => {
    // Return new optimistic state
  }
);
```

### Step 5: Replace react-helmet

```jsx
// Remove from package.json
npm uninstall react-helmet

// Replace in components
- <Helmet><title>Page</title></Helmet>
+ <title>Page</title>
```

|---------|----------|----------|
| Form handling | Manual | Built-in Actions ✅ |
| Optimistic UI | Manual | useOptimistic ✅ |
| Document metadata | Library needed | Native support ✅ |
| forwardRef | Required | Optional ✅ |
| Ref cleanup | Manual | Return function ✅ |

## When to Upgrade?

**Upgrade if:**
- ✅ You build lots of forms
- ✅ You need optimistic UI
- ✅ You want less boilerplate
- ✅ You're starting a new project

**Wait if:**
- ⏸️ Your dependencies don't support React 19 yet
- ⏸️ You use legacy APIs (string refs, old context)
- ⏸️ Your app is stable and has no performance issues
